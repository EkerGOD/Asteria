## 当前版本 v0.8.2 — 体验笔记

### Bug
-

### UI / 交互不合理
- 折叠按钮分布问题，左侧折叠按钮应该位于左侧，右侧折叠面板应该位于右侧
- 左侧折叠面板图表错误，为上侧折叠面板图表
- 设置按钮是太阳标志，不是齿轮标志，会导致误解
- 打开文件的之后会出现一个文件状态栏，位于顶端，但是我认为应该将文件状态栏修改到中央的位置，也就是中心Editor界面的上方，并随着左右侧栏的展开折叠自动变化。
- 当右侧栏切换到chat目录的时候，上下之间的位置可以通过将鼠标放置在其之间的分界线，点击进行调整，而不是现在这样固定死
- 设置界面的Providers和Diagnositics还只是简单吧之前的页面粘贴过来，没有考虑页面的适配，导致非常的拥挤，我认为页面应该采用竖排的结构，每一个不同的这只section用标题和边框进行区分
- EDIT PROVIDER栏位应该在点击对于Provider的Edit按钮的时候才会出现，而不是现在这样一直显示，因此要给每个Provider添加一个Edit按钮，点击New按钮的时候，才会出现NEW PROVIDER界面。DESKTOP这样总览的数据界面应该放在最顶上。
- Diagnostics页面也要调整布局，现在非常的拥挤

### 想要的功能
- 现在前端还只是展示界面，还没有和后端进行衔接
- 实现中央的编辑器功能
- 可以实现打开仓库，显示文件树

### 其他想法
- 前端有了较大的变化，要检查是否有冗余的代码，前端是否适配后端接口，接口设置是否合理，是否需要调整。
